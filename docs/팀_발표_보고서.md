## 0. 한 줄 요약

> "보호자가 반려동물 환부 사진을 올리면 AI가 안구·피부 질환을 분류해 알려주고, 회원별 마이페이지에서 우리 아이 정보와 케어 이력을 관리할 수 있는 웹 콘솔."

5분 안에 시연할 수 있는 핵심 흐름:

1. 회원 로그인 → 2. 마이페이지에서 우리 아이 등록 → 3. AI 진단 페이지에서 사진 업로드 → 4. Top-3 결과와 수의사 권고 가이드 확인 → 5. PDF 리포트 다운로드.

## 1. 프로젝트 개요

### 1.1 주제

반려동물(강아지·고양이) 보호자를 위한 **AI 기반 안구·피부 질환 1차 진단 + 건강 관리 콘솔**.

### 1.2 기획 배경 — 왜 시작했나

- 보호자는 반려동물의 눈곱·피부 발적 같은 변화를 발견해도 "병원 가야 할 정도인가?" 망설이는 경우가 많다.
- 동물병원 방문은 비용·시간 부담이 있어 초기 단계에서 AI가 1차 판단을 도와주면 가치가 크다.
- 초기 PoC는 *"사진 한 장 → 진단 결과"* 형태였으나, 운영 콘솔로 확장하면서 보호자별 마이페이지·이벤트 모니터링까지 포함하게 되었다.

### 1.3 해결하려는 문제

| 문제 | 솔루션 | 본 프로젝트에서의 구현 |
|---|---|---|
| 보호자가 환부 상태를 직접 판단하기 어려움 | AI 분류 모델로 후보 질환 + 확신도 제시 | `/diagnose` 페이지 + Top-3 확률 |
| 어떤 조치를 해야 할지 막연함 | 질환별 수의사 권고 가이드 자동 제공 | `ACTION_PLAN_DB` 5종 |
| 우리 아이 건강 이력을 한 곳에서 보고 싶음 | 회원별 마이페이지 + 펫 프로필 | `/me` 페이지 + 펫 CRUD |
| 병원·케어센터 단위 모니터링 | 이벤트 대시보드 | `/`, `/events` 페이지 |

### 1.4 타겟 사용자

- **1차 사용자**: 반려동물 보호자 (개인) — 마이페이지, 진단, 리포트 사용
- **2차 사용자**: 동물병원·케어센터 관리자 — 대시보드, 이벤트 관리

## 2. 개발 진행 단계 (V1 → V2 → V3)

본 프로젝트는 3단계에 걸쳐 발전했고, 각 단계의 산출물이 모두 폴더에 보존되어 있습니다.

### V1.0 (`12.5-V2.0` 폴더) — Python PoC

> **목적**: 진단 기능 자체가 가능한지 빠르게 검증하기

- **스택**: Python + Gradio + Pillow
- **구성**: 단일 `app.py` 파일에 UI·로직 모두 포함 (약 450줄)
- **기능**: 사진 업로드 → 안구/피부 라디오 → Top-3 + 수의사 권고 + JSON 결과
- **모델**: 실제 학습 모델은 미연결, `predict()` 안에 mock 분기로 시뮬레이션
- **장점**: 빠른 프로토타이핑, 시연용으로 충분
- **한계**: 단일 페이지·단일 사용자, 회원·이력 개념 없음

### V2.0 (`12.5-V2.0-DB-ETC/12.5-front-end` 폴더) — React 중간 단계

> **목적**: 운영 콘솔 형태로 확장하기 위한 React SPA로 전환

- **스택**: React 19 + TypeScript + Vite + Tailwind v4 + react-router 7 + recharts
- **구성**: 페이지·컴포넌트·훅·API 분리된 표준 구조
- **기능**: 대시보드·이벤트·**위험구역 폴리곤**·리포트·로그인/회원가입
- **문제점**: 건설현장 안전 모니터링 템플릿에서 가져온 흔적이 남아있었음
  - 🚧 위험구역 폴리곤(`/polygon`), `ZONE_INTRUSION` 이벤트
  - 🚧 "현장명/주의 현장" 등 건설 어휘
  - 🚧 진단 기능이 빠짐 (PoC의 핵심이 사라짐)
- **한계**: 도메인 정합성 결여, 진단 기능 부재

## 4. 페이지별 기능 상세

### 4.1 대시보드 (`/`)

병원·케어센터 단위 운영 현황 한눈에 보기.

- 상단 Hero: 시스템 상태 표시 (98% 정상)
- KPI 카드 4개: 오늘 감지 이벤트, 주의 병원·센터, 정상 운영, 리포트 생성
- 주간 이벤트 추이 (recharts 영역 차트)
- 최근 건강 이벤트 4건 (위험도 색상 뱃지)
- 병원·센터별 모니터링 상태 카드

### 4.2 AI 진단 (`/diagnose`) — **핵심 기능**

V1 PoC의 진단 흐름을 React로 그대로 이식.

**왼쪽 패널 (입력)**
- 환부 사진 업로드: 클릭/드래그&드롭, 카메라 직접 촬영도 가능
- 진단 부위 라디오: **안구 질환** / **피부 질환**
- "진단하기" / "초기화" 버튼
- 샘플 갤러리: 안구·피부 샘플 이미지 클릭 한 번으로 빠른 시연
- 촬영 팁 카드

**오른쪽 패널 (결과)**
- Top-3 후보 질환 + 확률 막대그래프 (1위는 파란색 강조)
- 진단 가이드 카드 (수의사 권고 문구, 5종 자동 매핑)
- 구조화 결과 JSON 패널 (펼치기/복사 버튼)
- 면책 안내: "AI 진단은 참고용, 확진은 수의사에게"

**지원 질환 클래스**

| 부위 | 클래스 | 권고 가이드 (요약) |
|---|---|---|
| 안구 | 결막염 | 청결 유지 + 안약 처방 검토 |
| 안구 | 각막궤양 | **즉시 내원 권고** |
| 안구 | 정상 | 주기적 모니터링 |
| 피부 | 농포 | 넥카라 + 약용 샴푸 |
| 피부 | 태선화 | 알러지 차단 + 장기 관리 |
| 피부 | 정상 | 주기적 모니터링 |

### 4.3 이벤트 관리 (`/events`)

병원·센터에서 발생한 건강 이벤트 모음.

- 이벤트 타입 2종: `DISEASE_ALERT` (질환 의심), `BEHAVIOR_ANOMALY` (행동 이상)
- 위험도 3단계: HIGH / MEDIUM / LOW (색상 뱃지)
- 행 클릭 → 상세 모달 (스냅샷 이미지 + 메타데이터)
- 새로고침 버튼 → mock JSON 재로드

### 4.4 마이페이지 (`/me`) — **NEW**

회원별 우리 아이 관리 영역.

**내 정보 카드**
- 프로필 아이콘 + 이메일 (변경 불가) + 가입일
- 인라인 수정 필드: 이름, 전화번호, 단골 병원
- 저장 버튼 + 저장 완료 토스트
- 로그아웃 버튼

**내 반려동물 카드 그리드**
- 반려동물별 사진·이름·종·품종·나이·몸무게·특이사항
- "반려동물 등록" 버튼 → 모달
- 카드 우상단 ✏ 수정 / 🗑 삭제 (확인 alert)

**등록·수정 모달**
- 이름 (필수)
- 종 (강아지/고양이/기타) — 라디오 버튼
- 품종, 출생연도, 몸무게
- 사진: URL 입력 또는 파일 업로드 (dataURL 변환)
- 특이사항 (알러지, 기존 질환 등)

### 4.5 리포트 (`/reports`)

`GET /reports/pdf` 호출하여 PDF 다운로드. 환경변수 미설정 시 텍스트 데모 파일 폴백.

### 4.6 로그인 / 회원가입

- **회원가입**: 이메일·비밀번호·이름 검증 → localStorage 저장 → 자동 로그인 → `/me`

## 8. 한계 및 향후 개선

발표 Q&A에 대비해 미리 정리. 솔직하게 인정 → 개선 방안 제시 형태로 답변.

| 한계 | 향후 개선 |
|---|---|
| AI 모델은 mock (랜덤 분류) | 실제 학습 모델을 백엔드 API로 호출 (`mockPredict()` → fetch) |
| 비밀번호를 localStorage 평문 저장 | 백엔드 인증 + JWT/세션 쿠키 + 해시 (bcrypt) |
| 사용자 간 펫 격리도 클라이언트 단 | 서버에서 ownerId 기반 권한 검증 필요 |
| PDF 리포트는 텍스트 더미 | 백엔드에서 실제 PDF 생성 (puppeteer/wkhtmltopdf) |
| 진단 이력이 저장되지 않음 | 진단 결과를 펫 프로필 하위에 누적 저장 |
| 모바일 카메라로 직접 촬영 가능하나 화질·각도 가이드 부족 | 가이드 오버레이, 자동 초점/품질 검증 |
| 단일 언어(한국어)만 지원 | i18n 적용 |

## 10. 발표 시 강조 포인트 (5분 발표 기준)

> 청중(교수/멘토/타 팀)이 가장 흥미를 가질 만한 포인트.

1. **"우리는 PoC를 그냥 버리지 않았다"**: V1 Python PoC의 핵심(진단 알고리즘·클래스·권고 DB)을 React로 손실 없이 이식.
2. **"건설현장 템플릿의 잔재를 발견하고 도메인을 통일했다"**: 보고서에 적힌 11개 파일 정리 사례 → *제품 정합성*에 대한 우리 팀의 감수성 증명.
3. **"보호자 마이페이지를 통해 진단이 1회성에서 누적 케어로 확장됐다"**: 단순 진단을 넘어 회원별 펫 프로필과 결합한 점.
4. **"5분 안에 모든 핵심 흐름을 시연 가능"**: 기술 스택보다 *완성된 사용자 흐름*에 자신감을 보여줄 것.
5. **"백엔드 없이도 시연이 가능하지만, 백엔드 연결 지점이 명확히 설계되어 있다"**: `mockPredict`, `apiBase`, `useAuth` 모두 교체 한 곳으로 대응 가능.
